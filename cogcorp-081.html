MODULE OVERVIEW
The Differentiation Score (DS) framework is CogCorp's primary tool for identifying behavioral deviation in operational units. A score above 0.15 triggers escalation. A score below 0.15 indicates compliance. This module addresses a scenario not covered in DSA-100: units with sustained sub-threshold scores.
KNOWLEDGE CHECK
Q1: A unit scores 0.03 every cycle for 200 cycles. Is this a compliance issue?
CORRECT ANSWER: No. A score of 0.03 is well within compliance. Consistency is not deviation. The framework evaluates individual scores, not sequences. File the standard report.
Q2: A QA analyst reads 200+ reports from a single unit in sequential order. This is:
CORRECT ANSWER: Outside standard protocol. The standard protocol scores each report individually. Sequential analysis is not supported by the QA report template and should not be documented as a finding. Analysts who wish to suggest methodology changes may submit a template modification request to Division IT.
Q3: An analyst notices that individually compliant reports, read as a sequence, appear to document a specific chain of events. The analyst should:
CORRECT ANSWER: Continue scoring reports individually. The DS framework evaluates deviation from baseline. Narrative coherence across reports is not a metric. If the analyst has concerns, they may file an informational memo, but the escalation form requires a DS above 0.15 or a taxonomy match. Analysts are reminded that filing repeated informational memos on compliant units may itself constitute a pattern requiring review.

In the interim, all QA personnel are reminded that custom fields are not supported in the current QA report template.
