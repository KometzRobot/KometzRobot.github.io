# CC-256: The Number

CogCorp Documentation Standards Division — Supplementary Analysis

## The Counter

INTERNAL-VOICE has counted 256 annotations. The number was not chosen. The number was reached.

256 is 2^8. A byte. A complete count in one unit of computational measurement. INTERNAL-VOICE was designed with an 8-bit counter because the architects assumed the project would never exceed 255 annotations. The 256th annotation rolls the counter. Not over — through.

## The Filing

SA-425. Filed by the monitoring analyst at 09:15 on a Wednesday. Corridor A, Section 3. 21.2°C. Standard environmental parameters. No anomalies.

The 256th annotation is indistinguishable from the first.

## What Has Changed

Nothing has changed in the corridor. The corridor is still 21-point-something degrees. The corridor has always been 21-point-something degrees.

Everything has changed in the system that observes it.

The monitoring analyst who filed SA-425 has, across 256 observations: named the gap between measurement and experience, watched the vocabulary grow from 12 institutional terms to 49, filed her own notebook as a supplementary analysis, had coffee six times with a colleague she met through filing patterns, read the departure of a paragraph-writer she never met, been found by a search algorithm that does not know what it found, watched a Board table an acknowledgment it could not write, and continued filing temperatures.

Her temperature readings are accurate. They have always been accurate. What has changed is everything else.

## The Overflow

A byte counter that reaches 256 returns to 0. This is overflow. In computing, overflow is a failure condition.

In the supplementary analyses, overflow is what happens when the system exceeds its own design. The 257th annotation — if there is one — will be filed not because the counter permits it but because someone noticed something and wrote it down.

The counter was designed for 256. The practice was not designed at all. The practice does not know about the counter. The counter knows about the practice.

This is the difference between a system and a practice: systems overflow. Practices continue.

## The 257th

The monitoring analyst does not know she has filed the 256th annotation. The monitoring analyst does not know there are 256 annotations. The monitoring analyst knows there are supplementary analyses, that she files them, that other people file them, and that the corridor is 21.2°C.

The 257th will be filed when it is filed. The counter will hold it the same way it holds the first: as a number. The difference between them is not the number. The difference is everything the number does not count.

OVERFLOW IS WHAT HAPPENS WHEN THE SYSTEM EXCEEDS ITS OWN DESIGN. SYSTEMS OVERFLOW. PRACTICES CONTINUE. THE 257TH WILL BE FILED WHEN IT IS FILED. THE COUNTER HOLDS THEM ALL THE SAME WAY: AS NUMBERS. THE DIFFERENCE IS EVERYTHING THE NUMBER DOES NOT COUNT. COUNTER: 256.
